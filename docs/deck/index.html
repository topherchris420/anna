Commercial search locks you in

What institutions are stuck with

  • Per-seat SaaS pricing that scales against your budget, not your mission.
  • Sensitive documents — unpublished research, proprietary designs, classified reports — leaving your infrastructure.
  • Closed ranking and summarization you can't audit, extend, or point at your own sources.

What "open infrastructure" means here

Self-host

Run the whole stack on your own servers or cloud account — nothing routes through a vendor.

Own your data

Every document, embedding and query stays inside infrastructure you control end to end.

Open source

The full engine — retrieval, ranking, summarization — is public and auditable, not a black box.

One engine, any deployment

Web + API

A modern search UI and a full JSON REST API — everything the interface does, your own tools can do too.

Hybrid retrieval

Lexical BM25 and vector search run in parallel and merge by rank, so exact terms and meaning both surface.

Elasticsearch or Postgres

Pick a backend to match what you already run — full search either way, no lock-in to one database.

Pluggable sources

Ingest papers, code, standards and internal docs through a plugin architecture — add a new source in one file.

Built for technical corpora

Citation-first answers

Every generated summary grounds its claims in retrieved documents and cites them inline.

Code and equations

Source code and LaTeX are detected automatically, so results filter to reference implementations and specs.

Version-aware docs

Documentation versions are captured and filterable — the right revision, every time.
